# Audit of global `window.ketcher` usage and related singleton assumptions

This audit covers the code paths that still depend on a page-global Ketcher instance or an equivalent singleton editor assumption. It is intended to support issue [#6506](https://github.com/epam/ketcher/issues/6506) and the macromolecules multi-instance bug tracked in [#6989](https://github.com/epam/ketcher/issues/6989).

## Scope

Searches were run across the repository source, examples, tests, and docs with `dist`, `node_modules`, and `.git` excluded.

Patterns audited:

- direct global access: `window.ketcher`
- singleton editor access: `CoreEditor.provideEditorInstance()`
- global mode flag: `window.isPolymerEditorTurnedOn`
- global typings that advertise those globals

## Summary

| Pattern                              | Occurrences | Files | Assessment                                                                                            |
| ------------------------------------ | ----------: | ----: | ----------------------------------------------------------------------------------------------------- |
| `window.ketcher`                     |         100 |    30 | Direct page-global editor access; only one instance can be the active global at a time.               |
| `CoreEditor.provideEditorInstance()` |         152 |    39 | Implicit singleton editor lookup; many macromolecule flows read whichever editor was registered last. |
| `window.isPolymerEditorTurnedOn`     |          28 |     6 | Global mode switch shared by all editor instances; one editor can change another editor's behavior.   |

## Recommended refactoring sequence

1. **Break the hidden editor singleton first.** Replace `CoreEditor.provideEditorInstance()` with instance-scoped access based on `ketcherId` or explicit dependency passing.
2. **Move macro/micro mode off `window`.** Every API/render/change-handling branch should read mode from the current editor instance.
3. **Remove runtime fallbacks to `window.ketcher`.** Rendering, clipboard error handling, and logging should all use injected instance dependencies.
4. **Update public examples and README.** Stop teaching `window.ketcher = ketcher` as the default integration pattern.
5. **Refit autotests around instance-local handles.** After product code is instance-safe, tests should wait for and address a specific editor instead of a page-global singleton.
6. **Tighten typings last.** Once compatibility shims are no longer required, remove or deprecate the global declarations so new code cannot regress.
